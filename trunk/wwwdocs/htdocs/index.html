What is GPUTILS ?

GPUTILS is a collection of tools for the Microchip (TM) PIC microcontrollers. It includes gpasm, gplink, and gplib. It's distributed under the terms of the GNU Public License. See the gnupic page for other GNU PIC tools such as gpsim, a gputils compatible simulator.

Who is GPUTILS ?

Currently, it is primarily supported by Craig Franklin and Scott Dattalo.

A Brief History

In 1997, James Bowman started gpasm. The objective was to create an open source replacement for Microchip's MPASM. During this time many people contributed to gpasm.

    Scott Dattalo
    James Cameron
    Stefan Petersen
    Eric Smith
    Alessandro Zummo
    Andrew Burgess
    Bruce Stough
    Tommy Thorn
    Charlie Krauter

At the end of 2000, Craig Franklin became the primary gpasm developer. Through most of 2001 the focus was compatiblity with MPASM. This was made much easiler by a contribution of MPASM test files to the project by Microchip. Also during this time, work started on relocatable object tools for gpasm. 

In 2002, A new package, gputils, containing gpasm and the new relocatable object tools was created.

What Platforms are Supported ?

Linux (and other unices), Mac OS X, and MS-Windows are supported. GPUTILS will compile natively on Linux using gcc. Mac OS X users will need the December 2001 Developer Tools. Windows 98/NT users will need MinGW32 if they wish to compile the tools.

Users have ported gputils to many other platforms. A few are listed in the downloads section. If you have generated a binary of gputils please notify the maintainer of this website, so that it can be added.

Downloading GPUTILS

The GPUTILS source tree is currently located at SourceForge.

As of January 20, 2002 - the current gputils release version is v0.10.0

    GPUTILS v0.10.0 Source (in gzipped tar format) [430K]

    GPUTILS v0.10.0 Linux i386 Binary (in rpm format) [263K]

    GPUTILS v0.10.0 Win32 i386 Binary (in zip format) [453K]

Older gputils and gpasm releases are available.

Binaries are also generated by thrid parties. Some of these binaries are for the old gpasm package.

    GPASM Debian Linux

    GPASM FreeBSD

    GPASM NetBSD

    GPASM RISC OS

Work continues in CVS. If you have CVS capability, the following will get you the latest sources.

cvs -d:pserver:anonymous@cvs.gputils.sourceforge.net:/cvsroot/gputils login

The password is empty and pressing [ENTER] will get you in.

The following command will checkout all of the sources and place them in a gpasm subdirectory.

cvs -z3 -d:pserver:anonymous@cvs.gputils.sourceforge.net:/cvsroot/gputils co gputils

You can browse our CVS tree here.

SourceForge has further documentation on accessing a CVS tree here.

Test Files

Microchip provided the project with a set of test files that they use to test MPASM for each release. These files were reformatted, some typos were fixed, and scripts to run the tests were generated. The files and scripts are located in cvs and can be accessed here.

For anyone interested, the original unaltered files are available in this archive. For information about the archive please read this file. These files were provided by Microchip without any warranty or any level of support. If you have a question about the files ask the current gputils project administrator, not Microchip.

Recent NEWS

January 20, 2002 -- gputils-0.10.0 Released

This release is basically gpasm-0.9.14 with bug fixes. Additionally, there are some new tools. However, these new tools are not complete, so they are not installed. Please see the ChangeLog for details on the bug fixes.

January 13, 2002 -- gputils imported to cvs

The new package has been imported to cvs. The first release should occur in the coming weeks.

December 12, 2001 -- gputils package created

A new linker and lib tool are under development. These new tools along with gpasm will be distributed in a new package gputils, GNU PIC Utilities.

Support for GPUTILS

GPUTILS comes with some documentation and examples. When that isn't enough, you can find help in the following places:

  • User Manual - for information on the tools in Acrobat format. You might also want to take a look at Microchip's MPASM manual.

  • Mailing list - if you have a general question about PIC micro controllers or the tools. Subscribe to the GNUPIC mailing list by sending a message to:

    gnupic-subscribe@linuxhacker.org

    with the word subscribe in the body.

    gnupic@linuxhacker.org is the address of the list.

    There's also the gnupic archive.

  • Bug Reporting - if you have a problem using GPUTILS, we need to hear about it.

  • Feature Requests - if you would like to see a new feature.

  • Source Forge Website - for further information.

  • GPSIM Information - for information on the gputils compatible simulator.
